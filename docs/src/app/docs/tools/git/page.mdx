## Usage Examples

### Status and Changes

```
what's the git status?
```

```
show me what I've changed
```

```
show the diff for src/index.ts
```

### Committing

```
commit all my changes with a good message
```

```
stage and commit the README changes
```

### History

```
show the last 5 commits
```

```
who wrote this line in utils.ts?
```

### Branches

```
create a new branch called feature/auth
```

```
switch to the main branch
```

```
merge feature/auth into main
```

## Permissions

- **Read operations** (status, diff, log) — Auto-approved
- **Write operations** (commit, push, merge) — Requires confirmation

---

## Best Practices

1. **Review diffs** before committing
2. **Use descriptive commit messages**
3. **Check status** before major operations
