# Doctor Command

The `doctor` command diagnoses your Helios installation and configuration.

---

## Usage

```bash
helios doctor
```

---

## What It Checks

| Check | Description |
|-------|-------------|
| API Keys | Configured provider keys |
| Model Access | Can connect to AI providers |
| Tools | All tools loaded correctly |
| MCP | MCP servers connected |
| Permissions | File system access |

## Common Issues

### No API Key Configured

```
‚úó No API keys configured
```

**Fix:** Set at least one API key:

```bash
helios config set OPENROUTER_API_KEY sk-or-...
```

### Model Access Failed

```
‚úó Model Access: Connection failed
```

**Fix:**
1. Check your internet connection
2. Verify API key is valid
3. Check provider status page

---

## Troubleshooting Workflow

1. Run `helios doctor`
2. Fix any `‚úó` items
3. Run `helios doctor` again
4. Start using Helios
